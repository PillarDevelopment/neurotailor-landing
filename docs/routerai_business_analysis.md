## EXECUTIVE SUMMARY

RouterAI — это платформа-агрегатор, предоставляющая единый API-доступ к 100+ моделям искусственного интеллекта (ChatGPT, Claude, Gemini, Grok и др.) с оплатой в рублях и работой без VPN на территории РФ. Целевая аудитория: разработчики, стартапы, корпоративные клиенты в России и СНГ, которым требуется стабильный доступ к AI-моделям без валютных и технических барьеров.

**Ключевые преимущества:** 
- Единый API для всех моделей
- Легальная оплата в рублях
- Работа без VPN
- Автоматическая отказоустойчивость

**Основные вызовы:**
- Высокая конкуренция (ProxyAPI, ChadGPT)
- Зависимость от иностранных провайдеров
- Необходимость постоянного обновления каталога моделей

## 1. БИЗНЕС-МОДЕЛЬ

### 1.1. Ценностное предложение (Value Proposition)

**Для разработчиков и стартапов:**
- **Единый интерфейс:** Один API-ключ, один эндпоинт вместо интеграции с 5+ различными провайдерами
- **Быстрое тестирование:** Переключение между моделями простой заменой параметра `model` в запросе
- **Экономия времени:** Экономия недель разработки на поддержке различных API

**Для корпоративных клиентов:**
- **Централизованное управление:** Единая точка контроля доступов, расходов и лимитов
- **Прозрачная аналитика:** Отслеживание расходов каждого сотрудника в реальном времени
- **Юридическая прозрачность:** Работа по договору, закрывающие документы, ЭДО
- **Соответствие ФЗ-152:** Модели, размещенные на территории РФ

**Для всех сегментов:**
- **Отсутствие барьеров входа:** Нет необходимости в VPN, иностранных картах, телефонах
- **Прозрачное ценообразование:** Pay-as-you-go в рублях, без подписок и скрытых комиссий
- **Стабильность:** Автоматическая отказоустойчивость при недоступности провайдера

### 1.2. Монетизация

**Модель оплаты:** Pay-as-you-go (оплата по факту использования)

**Тарификация:** 
- На основе токенов (входящих/исходящих)
- Разные цены для разных моделей
- Примеры цен (за 1M токенов):
  - DeepSeek V3.2: 56₽/169₽ (вход/выход)
  - Claude Sonnet 4.5: 303₽/1515₽
  - GPT-5.2: 176₽/1414₽

**Способы пополнения:**
- **Физ. лица:** Карты российских банков, СБП
- **Юр. лица:** Оплата по счету, договор, закрывающие документы

**Нет:** минимальных платежей, подписок, скрытых комиссий

### 1.3. Целевая аудитория

**Первичные сегменты:**

1. **Инди-разработчики:**
   - Нужда: Быстрое тестирование разных моделей для MVP
   - Боль: Сложность интеграции нескольких API, необходимость VPN
   - Цитата с лендинга: _"Я могу тестировать гипотезы с разными моделями, просто меняя одну строчку в конфиге"_

2. **Стартапы и малый бизнес:**
   - Нужда: Быстрый вывод AI-продукта на рынок
   - Боль: Недели на поддержку разных API, отсутствие отказоустойчивости
   - Цитата: _"Переход на RouterAI сэкономил нам недели разработки"_

3. **Корпоративные клиенты (юрлица, ИП):**
   - Нужда: Централизованное управление AI-инфраструктурой
   - Боль: Десятки личных кабинетов, разрозненные счета, отсутствие юридически чистых документов
   - Цитата: _"Главное для нас — возможность работать с юрлицом и получать закрывающие документы"_

**Вторичные сегменты:**
- MarTech-компании
- B2B SaaS с AI-функциями
- Агентства и консалтинговые компании

### 1.4. Основные метрики (предположительно)

- **ARPU (Average Revenue Per User):** зависит от интенсивности использования, от нескольких сотен до десятков тысяч рублей/месяц
- **CAC (Customer Acquisition Cost):** вероятно низкий за счет органического трафика и word-of-mouth в dev-сообществе
- **Churn Rate:** потенциально низкий из-за lock-in эффекта (интеграция в продукт)
- **LTV:** высокий для B2B-клиентов с регулярным использованием

### 2.3. Скрытые боли (неочевидные)

1. **Fear of vendor lock-in:** Разработчики боятся привязаться к одному провайдеру AI
   - *Решение RouterAI:* Возможность смены модели "на лету"
   
2. **Неопределенность будущих цен:** Боязнь резкого роста стоимости API
   - *Решение RouterAI:* Прозрачные цены в рублях, pay-as-you-go

3. **Compliance риски:** Юридические риски использования иностранных сервисов
   - *Решение RouterAI:* Серверы в РФ, соответствие ФЗ-152

4. **Отсутствие SLA гарантий:** Простои без компенсаций
   - *Потенциальная слабость RouterAI:* На лендинге не упомянуты SLA

## 7. ФИНАНСОВЫЕ АСПЕКТЫ

### 7.1. Ценовая стратегия

**Модель:** Pay-as-you-go (usage-based pricing)

**Преимущества для клиента:**
- Нет lock-in через подписки
- Платишь только за использование
- Прогнозируемые расходы

**Преимущества для RouterAI:**
- Масштабируемая выручка
- Автоматическое увеличение ARPU с ростом использования
- Низкий churn (платишь только когда используешь)

### 7.2. Маржинальность (гипотеза)

**Модель заработка:**
1. Покупка токенов у провайдеров оптом
2. Добавление наценки
3. Продажа клиентам

**Предполагаемая наценка:** 10-30% над стоимостью провайдера

**Дополнительная ценность помимо цены:**
- Единый интерфейс (экономия времени разработки)
- Отказоустойчивость
- Управление командой
- Рублевые платежи без конвертации

**Возможности монетизации:**
- Premium support для enterprise
- SLA гарантии (за доплату)
- Dedicated capacity
- Custom integrations

### 7.3. Unit Economics (гипотетически)

**Для среднего B2B клиента:**
- Средний чек: 50,000₽/месяц
- Маржа: ~20% = 10,000₽
- Операционные расходы: ~3,000₽
- Чистая прибыль с клиента: ~7,000₽/месяц

**LTV расчет:**
- Средняя продолжительность жизни клиента: 24 месяца
- LTV: 7,000₽ × 24 = 168,000₽

**CAC:**
- Органический трафик + word-of-mouth: ~5,000₽
- LTV/CAC ratio: 33.6 (отлично, >3 считается хорошим)
